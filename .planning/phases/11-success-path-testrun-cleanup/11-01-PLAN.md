<objective>
Wire success-path TestRun CR cleanup through the metric plugin's `GarbageCollect` hook. When Argo Rollouts invokes `GarbageCollect(ar, metric, limit)` — which it does to enforce the measurement retention limit per metric — the plugin walks `ar.Status.MetricResults[*].Measurements[*].Metadata["runId"]` for the target metric and issues a cleanup Delete on the k6-operator TestRun CR for each runID. Grafana Cloud is a no-op (the Cloud API retains runs server-side). Cleanup errors are logged at Warn and never surface as `RpcError`.

Purpose: Close the success-path leak that Phase 08.3 intentionally deferred when `spec.Cleanup=post` was removed. Terminate/Abort paths already clean up via `StopRun`; now the success path (terminal `Passed`/`Failed` on the metric plugin) does too. Semantically-distinct from `StopRun` (which means "cancel"), so we add a separate `Cleanup(ctx, cfg, runID) error` method on the Provider interface rather than reusing `StopRun` — this keeps future providers (in-cluster k6 Jobs) free to differentiate the two (a Job provider might cancel with `kubectl delete --grace-period=0` for StopRun but reap with `ttlSecondsAfterFinished` for Cleanup). On the operator path, both happen to share the same delete path today, extracted into a private helper `deleteCR` for reuse.

Output: New `Provider.Cleanup` interface method with three concrete implementations (K6OperatorProvider = delete CR, GrafanaCloudProvider = no-op, Router = dispatch). `K6MetricProvider.GarbageCollect` becomes a real loop instead of the current `return types.RpcError{}` one-liner. 10 files modified; 3 test files gain new cases; interface contract preserved.
</objective>

**Design decisions (encoded here so executor does not revisit them):**

- **Add `Cleanup` method, do NOT reuse `StopRun`.** `StopRun` currently means "cancel an in-flight run" (called by Terminate/Abort). `Cleanup` means "release resources after a terminal state." They share the delete path on the operator backend TODAY, but the semantic distinction lets a future Job provider differentiate (cancel with `delete --grace-period=0`, cleanup with `ttlSecondsAfterFinished`). The ROADMAP plan-01 hint names `provider.Cleanup(ctx, cfg, runID)` — implement exactly that.

- **REQUIREMENTS.md states a fictional signature.** GC-01 quotes `GarbageCollect(ar *v1alpha1.AnalysisRun, metrics []*v1alpha1.Measurement) error`. That signature does not exist in Argo Rollouts v1.9.0 — the real interface is `GarbageCollect(*v1alpha1.AnalysisRun, v1alpha1.Metric, int) types.RpcError` (verified in `~/go/pkg/mod/github.com/argoproj/argo-rollouts@v1.9.0/utils/plugin/types/types.go:62` and `metricproviders/plugin/rpc/rpc.go:121`). We honor the REAL signature because the plugin gRPC contract is versioned and a hard constraint (PROJECT.md). To recover the list of runIDs to clean up, we walk `ar.Status.MetricResults` for the entry matching `metric.Name`, then iterate that entry's `Measurements`. This mirrors how Argo's own `JobProvider` implements `GarbageCollect` (by walking Kubernetes resources associated with `run.UID` via label selector). We pick the "walk `ar.Status.MetricResults[*].Measurements[*].Metadata[\"runId\"]`" approach because our runIDs are not deterministically derivable from `ar.UID` (they include a timestamp hash — see `testRunName` in `internal/provider/operator/testrun.go:53`), so the label-selector approach would require a new `list` RBAC verb on testruns (already granted, but `Measurements` is authoritative and simpler).

- **`limit` parameter is ignored** for the cleanup decision (not for the retention enforcement — that is Argo's business). Argo Rollouts only calls `GarbageCollect` when `len(measurements) > limit`, and after our call returns, it trims the slice to the last `limit` measurements. We delete the CR for EVERY measurement we receive in `ar.Status.MetricResults`, because by the time Argo trims the slice, the earlier measurements are the ones being discarded. Note: some of those earlier measurements may correspond to non-terminal runs that were later re-run — but each measurement has its own unique runID stored in Metadata, and the TestRun CR for a non-terminal run will have been replaced by Resume/Run cycles. Deleting by runID is safe: if the CR is already gone (reclaimed, or the run is still in flight), `k8serrors.IsNotFound` makes this a no-op.

- **Grafana Cloud is a no-op.** The Grafana Cloud k6 API retains completed runs server-side (users view historical runs via the Grafana dashboard); the plugin has no client-side resources to release. `GrafanaCloudProvider.Cleanup` returns `nil` and logs at `slog.Debug` (not Info, to avoid log spam in the common case).

- **GC-03 error swallowing.** `K6MetricProvider.GarbageCollect` MUST return `types.RpcError{}` (empty) even if some or all Cleanup calls fail. Per-call errors are logged at `slog.Warn`. This matches how `Terminate` (metric.go:197-202) already swallows StopRun errors.

- **Concurrency/ordering.** Cleanup calls run sequentially in a simple for-loop. We do not parallelize because (a) GarbageCollect is called with `measurementsLimit` defaulting to 2 per AR (Argo default), so the slice is tiny in practice, and (b) goroutine + errgroup machinery adds complexity with no latency benefit — the whole GC pass happens asynchronously on the controller side.

- **Context budget.** Each Cleanup call uses a separate `context.WithTimeout(ctx, providerCallTimeout)` derived from a fresh `context.Background()` (not from an ambient GarbageCollect ctx — the RPC interface provides none). Matches the per-call-timeout pattern in Run/Resume/Terminate.

- **No changes to `cmd/metric-plugin/main.go` or `cmd/step-plugin/main.go`.** The Router is constructed the same way; the new interface method is present on the already-registered `cloudProvider` and `operatorProvider`.

    ```go
    // GarbageCollect is called by the Argo Rollouts analysis controller when the
    // number of measurements for a metric exceeds the retention limit (see
    // analysis/analysis.go:793 in argo-rollouts v1.9.0). We use this hook as the
    // success-path cleanup trigger for k6-operator TestRun CRs created by Run/Resume.
    //
    // Walks ar.Status.MetricResults for the entry matching metric.Name, iterates
    // its Measurements, extracts Metadata["runId"] per measurement, and dispatches
    // provider.Cleanup for each non-empty runID. The Router resolves the correct
    // provider backend based on cfg.Provider — grafana-cloud is a no-op, k6-operator
    // issues an idempotent Delete on the TestRun CR.
    //
    // Per GC-03: cleanup errors are logged at slog.Warn with structured fields
    // (runId, metric, analysisRun, namespace, error) but NEVER returned as RpcError.
    // This mirrors how Terminate swallows StopRun errors.
    //
    // The `limit` parameter is used by Argo Rollouts after our return to trim
    // the Measurements slice; we intentionally do not consult it here — we clean
    // up every runID in the slice. CRs for runIDs whose backing CR is already gone
    // (reclaimed, or the run is still in flight) are no-ops via IsNotFound handling
    // in the provider.
    func (k *K6MetricProvider) GarbageCollect(ar *v1alpha1.AnalysisRun, metric v1alpha1.Metric, _ int) types.RpcError {
        if ar == nil {
            return types.RpcError{}
        }

        cfg, err := parseConfig(metric)
        if err != nil {
            // Config parse failure at GC time is non-fatal: log and return empty RpcError.
            slog.Warn("garbage collect: parseConfig failed, skipping cleanup",
                "metric", metric.Name,
                "error", err,
            )
            return types.RpcError{}
        }
        populateFromAnalysisRun(cfg, ar, "metric.GarbageCollect")

        for _, result := range ar.Status.MetricResults {
            if result.Name != metric.Name {
                continue
            }
            for _, m := range result.Measurements {
                runID := m.Metadata["runId"]
                if runID == "" {
                    continue
                }
                ctx, cancel := context.WithTimeout(context.Background(), providerCallTimeout)
                if cleanupErr := k.provider.Cleanup(ctx, cfg, runID); cleanupErr != nil {
                    slog.Warn("failed to cleanup run during garbage collect",
                        "runId", runID,
                        "metric", metric.Name,
                        "analysisRun", cfg.AnalysisRunName,
                        "namespace", cfg.Namespace,
                        "error", cleanupErr,
                    )
                }
                cancel()
            }
        }
        return types.RpcError{}
    }
    ```

    **Note the `cancel()` placement:** the context is cancelled at the END of each iteration to release resources between calls. Do NOT use `defer cancel()` inside the loop (defer-in-loop leaks until function return). This is a common Go pitfall the executor must get right.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| argo-rollouts controller → plugin binary | RPC over `net/rpc` / gob; controller supplies `*v1alpha1.AnalysisRun`, `v1alpha1.Metric`, and a measurement retention `int`. The plugin trusts the controller to pass well-formed types (gob-level type safety is enforced by the runtime). |
| plugin binary → kube-apiserver | Plugin issues `DELETE testruns.k6.io/<name>` via the in-cluster dynamic client. The cluster's ServiceAccount RBAC (see `examples/k6-operator/clusterrole.yaml`) already grants `delete` on `testruns` and `privateloadzones` — no RBAC change required. |
| User YAML (AnalysisTemplate) → plugin config | `metric.Provider.Plugin[pluginName]` is unmarshaled into `PluginConfig`. The `runId` field is not user-supplied — it is injected by Run/Resume after `TriggerRun` returns. Cleanup consumes runIDs from `ar.Status.MetricResults[*].Measurements[*].Metadata`, which Argo Rollouts persists from the plugin's own return values. |
