<objective>
Fix the Phase 10 e2e blocker: `buildTestRun` forwards `cfg.Parallelism` as `int32(0)` when the user omits `parallelism` in their plugin config, and k6-operator interprets `spec.Parallelism=0` as "paused" — runner pods never spawn and the AnalysisRun hangs forever. Default to `1` at the construction site.

Purpose: Make `parallelism` genuinely optional in the plugin config (as its JSON tag `parallelism,omitempty` and the existing "0 means unset" validation invariant already suggest). The fix is a three-line change inside a pure builder function, paired with targeted tests, Godoc, a README sentence, and one e2e regression guard.

Output: `buildTestRun` resolves `cfg.Parallelism == 0` to `1`; two new builder tests pin the behavior; Godoc on `PluginConfig.Parallelism` and `buildTestRun` document the default; README carries a one-line note; e2e has a dedicated test that exercises the full path with `parallelism` omitted. `ValidateK6Operator` and its two existing tests stay untouched.
