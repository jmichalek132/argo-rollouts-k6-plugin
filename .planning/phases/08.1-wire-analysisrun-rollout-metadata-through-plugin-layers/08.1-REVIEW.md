## Summary

The phase cleanly fixes the bug where `*v1alpha1.AnalysisRun` and `*v1alpha1.Rollout` were discarded at the plugin layer. The implementation is tight, faithful to the locked decisions (D-01..D-09), and the test coverage is genuinely strong.

Highlights:

- **Nil-pointer safety at the RPC boundary is correct.** `populateFromAnalysisRun` and `populateFromRollout` both guard against nil, log a warning, and fall through to user cfg without panic. Range over nil `OwnerReferences` is a Go no-op and needs no extra check.
- **Namespace precedence (D-01) is correct.** `cfg.Namespace == ""` is the only gate that lets the parent namespace win; user cfg always wins. `TestRun_UserNamespaceWinsOverAR` and `TestRun_UserNamespaceWinsOverRollout` directly exercise this.
- **Owner-reference walk (D-03) is correct.** The walk filters both `Kind == "Rollout"` AND `Controller != nil && *owner.Controller`. Non-Rollout kinds are skipped; non-controller Rollout refs are skipped. `TestRun_PrefersControllerRolloutOwnerRef` is a solid regression fixture — it proves first-match is NOT sufficient, only Controller==true wins.
- **D-07 precedence (AR > Rollout > none) is correct.** `parentOwnerRef` returns early on `AnalysisRunUID != ""`, else falls through to `RolloutUID`, else returns nil. `TestTriggerRun_AnalysisRunUIDWinsOverRolloutUID` and `TestTriggerRun_WithRolloutUID` both directly verify this branch structure.
- **Thread safety is not a concern.** `cfg` is created fresh in `parseConfig` via `var cfg provider.PluginConfig` and returned as `&cfg`. No cross-goroutine sharing of the mutated struct. `TestConcurrentSafety` runs 10 Resume goroutines successfully.
- **Logging hygiene is clean.** The new `slog.Info`/`slog.Warn` calls log only non-sensitive identifiers (runId, namespace, rollout/AR names, phase). `APIToken`, `StackID`, and `cfg.Env` are never logged.
- **Test fixtures match claimed behaviors.** `testAR` stamps `Controller=&true` by default; the multi-owner-ref fixture inlines both refs and asserts the correct one is chosen. `TestTerminate_PassesPopulatedConfigToStopRun` closes the MEDIUM verification gap for the stop path (R6 from prior codex review).

## Info

### IN-01: `populateFromAnalysisRun` log message is misleading when AR has a non-controller Rollout owner ref

**File:** `internal/metric/metric.go:246-249`
**Issue:** When an AnalysisRun has an `OwnerReferences` entry with `Kind == "Rollout"` but `Controller != true` (a malformed AR — should never happen in production since `NewControllerRef` always sets Controller, but possible in test fixtures or hand-crafted CRs), the walk falls through to the "standalone AnalysisRun (no Rollout owner ref)" warning. That label is misleading: the AR is not standalone; it just lacks the controller flag on its Rollout ref. An operator debugging this from logs would look for a missing owner ref when in fact the ref exists but is malformed.

Similarly, an AR with only non-Rollout owner refs (e.g., a Deployment owner) logs "no Rollout owner ref" — which is accurate but doesn't convey that other owner refs exist.

**Fix:** Either refine the warning message to distinguish the two cases, or include `ownerRefCount` as a slog field so the operator can tell at a glance whether the AR has zero refs or some refs without a valid controller Rollout:

```go
slog.Warn("AnalysisRun has no controller Rollout owner ref; RolloutName left empty",
    "analysisRun", ar.Name,
    "namespace", ar.Namespace,
    "ownerRefCount", len(ar.OwnerReferences),
    "phase", phase,
)
```

This keeps the D-04 semantics (empty RolloutName is acceptable) while giving the operator a signal to look at the AR's `OwnerReferences` when debugging a test fixture or a manual AR.

### IN-02: `populateFromRollout` does not defend against empty `rollout.Name` when `rollout.UID` is set

**File:** `internal/step/step.go:261-274`
**Issue:** `populateFromRollout` unconditionally copies `cfg.RolloutName = rollout.Name` and `cfg.RolloutUID = string(rollout.UID)` when `rollout != nil`. If an upstream caller constructs a malformed Rollout with an empty `Name` but a non-empty `UID` (should never happen for a real cluster object — K8s rejects nameless objects — but possible via a test double or a race between object creation and controller read), `parentOwnerRef` at `internal/provider/operator/testrun.go:88-95` will emit an OwnerReference with `UID=<something>` and `Name=""`. The testrun.go:75-78 caller invariant comment correctly states that the API server will reject this at Create time — so the failure mode is a loud error, not silent corruption. Still, this is detectable earlier.

**Fix:** Add a defensive check that treats empty Name as equivalent to nil for the RolloutUID path, mirroring the standalone-AR warning in the metric plugin:

```go
cfg.RolloutName = rollout.Name
if rollout.Name == "" {
    slog.Warn("Rollout has empty Name; skipping RolloutUID extraction",
        "phase", phase,
        "namespace", rollout.Namespace,
    )
} else {
    cfg.RolloutUID = string(rollout.UID)
}
if cfg.Namespace == "" {
    cfg.Namespace = rollout.Namespace
}
```

Low priority — the API-server rejection is sufficient defense today. Flagging for hardening awareness; out of scope as a must-fix.

### IN-03: `TestTriggerRun_WithRolloutUID` does not assert the `Namespace` or `Kind` scope of the Rollout ref

**File:** `internal/provider/operator/operator_test.go:347-379`
**Issue:** The test asserts `Kind=="Rollout"`, `Name=="my-app"`, `UID=="rollout-uid-777"`, and `APIVersion=="argoproj.io/v1alpha1"` — good. But it doesn't assert the `BlockOwnerDeletion` or `Controller` flags on the emitted OwnerReference. The current `parentOwnerRef` implementation leaves both unset (nil) — which is arguably correct for a plugin that doesn't want GC semantics it can't guarantee — but the test doesn't lock in the current behavior, so a future refactor could silently start setting `Controller=true` on the Rollout ref without any test catching it.

**Fix:** Add one assertion per ref field that `parentOwnerRef` intentionally leaves unset, to lock in the current contract:

```go
assert.Nil(t, ownerRefs[0].Controller, "parentOwnerRef must not set Controller on the Rollout ref (avoid adoption semantics)")
assert.Nil(t, ownerRefs[0].BlockOwnerDeletion, "parentOwnerRef must not set BlockOwnerDeletion")
```

Same applies to `TestTriggerRun_WithAnalysisRunUID` and `TestTriggerRun_AnalysisRunUIDWinsOverRolloutUID` for completeness. Minor — purely defensive test hardening.
