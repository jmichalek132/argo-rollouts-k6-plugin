**Create internal/provider/operator/exitcode.go**

Package `operator`. Contains:

1. Exit code constants (per RESEARCH.md, cited from k6 source):
```go
const (
    exitCodeSuccess          = 0
    exitCodeThresholdsFailed = 99
)
```

2. `exitCodeToRunState(code int32) provider.RunState` -- per D-05:
   - 0 -> provider.Passed
   - 99 -> provider.Failed (thresholds breached)
   - any other non-zero -> provider.Errored (script error, infra issue)

3. `stageToRunState(stage string) provider.RunState` -- per RESEARCH.md stage mapping:
   - "initialization", "initialized", "created", "started" -> provider.Running
   - "stopped" -> provider.Running (stopping but not finished)
   - "finished" -> provider.Running (IMPORTANT: returns Running because exit code check is needed to determine Passed/Failed -- this is the issue #577 workaround)
   - "error" -> provider.Errored
   - default (unknown) -> provider.Running (keep polling)

4. `checkRunnerExitCodes(ctx context.Context, client kubernetes.Interface, ns, testRunName string) (provider.RunState, error)`:
   - Label selector: `fmt.Sprintf("app=k6,k6_cr=%s,runner=true", testRunName)` (per RESEARCH.md, labels from k6-operator runner.go)
   - List pods with `client.CoreV1().Pods(ns).List(ctx, metav1.ListOptions{LabelSelector: selector})`
   - If len(pods.Items) == 0 -> return provider.Errored, fmt.Errorf("no runner pods found for TestRun %s", testRunName)
   - Iterate pods:
     - For each pod, check ContainerStatuses
     - If ANY container has Terminated == nil -> return provider.Running, nil (per Pitfall 2: pods not yet terminated, retry on next poll)
     - If ANY container has Terminated.ExitCode != 0 -> collect worst result using exitCodeToRunState
   - If all containers exit 0 -> return provider.Passed, nil
   - If any exit 99 -> return provider.Failed, nil (threshold failure takes precedence over Errored)
   - If any exit non-zero non-99 -> return provider.Errored, nil

   Exit code precedence logic: Errored > Failed > Passed. Use a helper or simple loop:
   ```go
   worst := provider.Passed
   for _, pod := range pods.Items {
       for _, cs := range pod.Status.ContainerStatuses {
           if cs.State.Terminated == nil {
               return provider.Running, nil // pod still running, retry
           }
           state := exitCodeToRunState(cs.State.Terminated.ExitCode)
           if state == provider.Errored {
               return provider.Errored, nil // worst case, return immediately
           }
           if state == provider.Failed {
               worst = provider.Failed
           }
       }
   }
   return worst, nil
   ```

   Logging: `slog.Debug("checked runner pod exit codes", "testRunName", testRunName, "namespace", ns, "podCount", len(pods.Items), "result", worst)`

Imports: `"context"`, `"fmt"`, `"log/slog"`, `metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"`, `"k8s.io/client-go/kubernetes"`, `"github.com/jmichalek132/argo-rollouts-k6-plugin/internal/provider"`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PluginConfig -> CR construction | User-provided config fields (parallelism, image, env) flow into Kubernetes CRD spec |
| k8s API -> pod listing | Pod status data from API server flows into pass/fail determination |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Denial of Service | config.go / ValidateK6Operator | mitigate | Validate parallelism >= 0 in ValidateK6Operator; upper bound enforcement deferred to cluster-level ResourceQuota (not plugin responsibility) |
| T-08-02 | Elevation of Privilege | testrun.go / buildTestRun | accept | Runner image comes from user config (AnalysisTemplate YAML). Plugin does not enforce image provenance -- this is the user's responsibility via admission controllers or OPA policies. Documented in RESEARCH.md security domain. |
| T-08-03 | Tampering | exitcode.go / checkRunnerExitCodes | accept | Pod status data comes from kube-apiserver which is trusted. Label selector is deterministic (not user-controlled). |
| T-08-04 | Information Disclosure | testrun.go / testRunName | mitigate | CR name uses SHA256 hash including namespace to prevent cross-namespace name collision. Hash includes timestamp for uniqueness. |
</threat_model>
