## Why

Phase 08.2's `parallelism=1` default let TestRun CRs actually run to completion instead of hanging paused. That success exposed a pre-existing bug: every TestRun set `spec.Cleanup=post`, which k6-operator reads as "delete this CR and its pods as soon as stage transitions to finished/error." The metric plugin's 10s Resume interval then raced against the operator's cleanup ÔÇö the plugin would try to `Get` the TestRun, see `NotFound`, and return an error. After `consecutiveErrorLimit=4` retries the AnalysisRun was marked `phase=Error` and the test failed.

The fix is three lines of production code (removing one struct field assignment); the test rename captures the intent shift; the diagnostic enhancement is worth keeping because it's what unstuck the investigation (the original dump only showed TestRuns and pods at teardown, which hid the fact that CRs had existed and self-deleted seconds earlier).
