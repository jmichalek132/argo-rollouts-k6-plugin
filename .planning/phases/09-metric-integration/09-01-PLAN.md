<objective>
Create the handleSummary JSON parsing engine for k6-operator metric extraction.

Purpose: The k6-operator provider's GetRunResult currently returns only State and ThresholdsPassed (from exit codes). This enables metric parity (METR-02) so users switch providers without rewriting AnalysisTemplates.

**PodLogReader interface (per RESEARCH Pattern 1, addresses Pitfall 3 -- fake clientset GetLogs returns constant "fake logs"):**
```go
type PodLogReader interface {
    ReadLogs(ctx context.Context, namespace, podName string, opts *corev1.PodLogOptions) (string, error)
}

type k8sPodLogReader struct {
    client kubernetes.Interface
}

func (r *k8sPodLogReader) ReadLogs(ctx context.Context, namespace, podName string, opts *corev1.PodLogOptions) (string, error) {
    req := r.client.CoreV1().Pods(namespace).GetLogs(podName, opts)
    stream, err := req.Stream(ctx)
    if err != nil {
        return "", fmt.Errorf("stream logs for pod %s/%s: %w", namespace, podName, err)
    }
    defer stream.Close()
    buf, err := io.ReadAll(stream)
    if err != nil {
        return "", fmt.Errorf("read logs for pod %s/%s: %w", namespace, podName, err)
    }
    return string(buf), nil
}
```

**findSummaryJSON function (per D-02, RESEARCH Pattern 3):**
Scan log output from the end to find a JSON object with `"metrics"` key. Implementation:
1. Split output into lines.
2. Walk backward from last line to find closing `}`.
3. Accumulate lines walking further backward counting brace depth until matching opening `{` found.
4. Attempt `json.Unmarshal` on the accumulated block into `k6Summary`.
5. Validate: parsed object must have non-nil `Metrics` map (rejects console.log JSON fragments that happen to parse but lack the `"metrics"` key).
6. Return nil, nil if no JSON found (graceful -- not an error per D-05).
7. Return nil, error only for malformed JSON (found what looks like JSON but it fails to parse).

**extractMetricsFromSummary function (per D-06, RESEARCH Example 2):**
```go
func extractMetricsFromSummary(summary *k6Summary) summaryMetrics {
    var m summaryMetrics
    // http_req_failed: rate metric
    if metric, ok := summary.Metrics["http_req_failed"]; ok {
        m.HTTPReqFailed = metric.Values["rate"]
    }
    // http_req_duration: trend metric
    if metric, ok := summary.Metrics["http_req_duration"]; ok {
        // P50: "med" key (NOT "p(50)") per Pitfall 1
        if v, ok := metric.Values["med"]; ok {
            m.HTTPReqDuration.P50 = v
        } else if v, ok := metric.Values["p(50)"]; ok {
            m.HTTPReqDuration.P50 = v
        }
        m.HTTPReqDuration.P95 = metric.Values["p(95)"]
        m.HTTPReqDuration.P99 = metric.Values["p(99)"] // 0.0 if not in summaryTrendStats
    }
    // http_reqs: counter metric
    if metric, ok := summary.Metrics["http_reqs"]; ok {
        m.HTTPReqs = metric.Values["rate"]
        m.httpReqsCount = metric.Values["count"]
    }
    // Derived: failed count for multi-pod aggregation
    if metric, ok := summary.Metrics["http_reqs"]; ok {
        m.httpReqFailedCount = m.HTTPReqFailed * metric.Values["count"]
    }
    return m
}
```

**aggregateMetrics function (per D-04, RESEARCH Pattern 4):**
For `len(metrics) == 0`: return zero-value summaryMetrics.
For `len(metrics) == 1`: return that single entry directly.
For `len(metrics) > 1`: weighted aggregation:
- `HTTPReqs = sum(m.HTTPReqs)` (sum of rates across pods)
- `totalCount = sum(m.httpReqsCount)` (total requests across all pods)
- `HTTPReqFailed = sum(m.httpReqFailedCount) / totalCount` (weighted average, guard div-by-zero)
- `HTTPReqDuration.P50 = sum(m.P50 * m.httpReqsCount) / totalCount` (weighted average)
- `HTTPReqDuration.P95 = sum(m.P95 * m.httpReqsCount) / totalCount`
- `HTTPReqDuration.P99 = sum(m.P99 * m.httpReqsCount) / totalCount`
