**Why this shape works:** after the initial (step-skipped) deploy reaches Healthy, patching the pod template triggers the canary steps. Step 1 (setWeight 20) is near-instant. Step 2 (step plugin) creates the Rollout-owned TestRun and starts polling. Step 3 (analysis) kicks off an AnalysisRun that references the AnalysisTemplate; the metric plugin on that AR creates the AR-owned TestRun. Both TestRuns coexist during the 120s k6 script window — plenty of headroom for the polling + kubectl-delete sequence.

**Alternative considered (rejected):** background analysis via `canary.analysis` (concurrent with all steps) instead of a dedicated analysis step. Rejected because it complicates AR name discovery (background analyses use generated names like `<rollout>-<revision>-<template>`) and the test's central assertion hinges on `kubectl delete analysisrun <known-name>`. A dedicated analysis step gives the AR a predictable name derivable from the Rollout's current step index (Argo names them `<rollout>-<revision>-<step-index>-<template>`). Even simpler: the test can list AnalysisRuns and pick the one whose `metadata.ownerReferences[0].kind == "Rollout"` — no name-guessing needed.

### `parentOwnerRef` behavior (from `internal/provider/operator/testrun.go:79-97`)

```go
// D-07 precedence (AR > Rollout > none)
func parentOwnerRef(cfg *provider.PluginConfig) []metav1.OwnerReference {
    if cfg.AnalysisRunUID != "" {                        // metric plugin path
        return []metav1.OwnerReference{{Kind: "AnalysisRun", Name: cfg.AnalysisRunName, UID: ...}}
    }
    if cfg.RolloutUID != "" {                            // step plugin path
        return []metav1.OwnerReference{{Kind: "Rollout", Name: cfg.RolloutName, UID: ...}}
    }
    return nil
}
```

- Metric plugin wires `cfg.AnalysisRunUID` via `populateFromAnalysisRun` (`internal/metric/metric.go`) → AR owner ref on the TestRun → this ref is the cascade hook.
- Step plugin wires `cfg.RolloutUID` via `populateFromRollout` (`internal/step/step.go`) and never sees an AR (step plugin has no AR context by design) → Rollout owner ref on the TestRun.

### New helper: `getTestRunsOwnedBy`

```go
// getTestRunsOwnedBy returns the names of TestRun CRs in `namespace` whose
// metadata.ownerReferences contain an entry matching ownerKind (and, if
// ownerName is non-empty, also matching ownerName). Used to distinguish
// AR-owned TestRuns from Rollout-owned TestRuns in the GC-cascade assertion.
//
// Implementation: kubectl get testruns -n <ns> -o json; walk items[].metadata.ownerReferences.
func getTestRunsOwnedBy(cfg *envconf.Config, namespace, ownerKind, ownerName string) ([]string, error) {
    cmd := exec.Command("kubectl", "--kubeconfig", cfg.KubeconfigFile(),
        "get", "testruns", "-n", namespace, "-o", "json")
    out, err := cmd.Output()
    if err != nil {
        return nil, err
    }
    var list struct {
        Items []struct {
            Metadata struct {
                Name            string `json:"name"`
                OwnerReferences []struct {
                    Kind string `json:"kind"`
                    Name string `json:"name"`
                } `json:"ownerReferences"`
            } `json:"metadata"`
        } `json:"items"`
    }
    if err := json.Unmarshal(out, &list); err != nil {
        return nil, err
    }
    var names []string
    for _, item := range list.Items {
        for _, ref := range item.Metadata.OwnerReferences {
            if ref.Kind != ownerKind {
                continue
            }
            if ownerName != "" && ref.Name != ownerName {
                continue
            }
            names = append(names, item.Metadata.Name)
            break
        }
    }
    return names, nil
}
```

// getTestRunStage returns status.stage of a named TestRun. k6-operator v1.3.x

**Design decisions (encoded here so executor does not revisit them):**

- **Single test function, not a table-driven suite.** The test orchestrates a stateful timing sequence (apply → wait → delete AR mid-run → assert cascade → assert survivor) that does not decompose cleanly into subtests. One test per Assess step is clearer than a shared harness.

- **Dedicated analysis step in `canary.steps[]`, not background `canary.analysis`.** Dedicated step gives the Rollout a predictable single AR per revision, and the step plugin + analysis run in sequence (step finishes, THEN analysis starts), but the step plugin blocks until its TestRun completes — which for a 120s script means the step TestRun and the analysis TestRun DO NOT overlap in time. **This breaks the test premise.** Fix: use **background analysis** via `canary.analysis.templates[]` at the `spec.strategy.canary` level, which starts the AnalysisRun concurrently with the canary steps. The step plugin's TestRun and the background AR's TestRun then coexist. See updated fixture note below.

  **Revised Rollout fixture shape (the one the executor MUST use):**
  ```yaml
  spec:
    strategy:
      canary:
        analysis:
          templates:
            - templateName: k6-operator-combined-e2e
          startingStep: 1        # start AR when the step plugin step begins
        steps:
          - setWeight: 20
          - plugin:
              name: jmichalek132/k6-step
              config:
                provider: k6-operator
                configMapRef: {name: k6-e2e-script-long, key: test.js}
                runnerImage: "grafana/k6:0.56.0"
                timeout: "5m"
          - setWeight: 100
  ```
  The `startingStep: 1` ensures the background AR launches AT THE SAME TIME as the step plugin step (both at index 1 after the setWeight 20 step). Both TestRuns become concurrent.

- **120s k6 `duration`, not `iterations`.** `iterations: 10` finishes in ~1 second; the existing short-script fixture is unusable here. `duration: "120s"` pins the run to a known wall-clock window. The AR-delete window (poll-for-running + delete + poll-for-cascade) fits easily inside 120s with budget to spare.

- **Wait for both TestRuns to reach `stage == "started"` BEFORE deleting the AR.** k6-operator v1.3.x enum (from `k6-operator/api/v1alpha1/testrun_types.go` + our own `internal/provider/operator/exitcode.go`): `initialization, initialized, created, started, stopped, finished, error`. There is NO `"running"` literal — the earlier draft's `"running"` guard would have timed out silently. `"started"` is the post-reconcile stage where runner pods are spawned and actively loading. Waiting for it confirms a subsequent disappearance is unambiguously kube-cascade-GC from our AR delete. The wait MUST fatal with `dumpK6OperatorDiagnostics` on timeout (do NOT proceed to delete if `started` is not observed).

- **Poll-based cascade assertion with a generous timeout.** kube-apiserver's cascading GC runs via the `garbage-collector` controller; latency depends on workqueue depth. 90s upper bound is conservative (typical latency on kind is 5-15s, but CI hosts can be slow). If the assertion times out, `dumpK6OperatorDiagnostics` captures full state.

- **Rollout-owned-survives assertion bounded by a small post-cascade wait.** Once `getTestRunsOwnedBy(cfg, ns, "AnalysisRun", "")` returns empty, wait 5-10s more and THEN assert `getTestRunsOwnedBy(cfg, ns, "Rollout", rolloutName)` is still non-empty. The extra wait catches any delayed cascade ripple that could plausibly (but should NOT) take the step TestRun with it.

- **Test is wall-clock expensive.** Budget ~5 minutes (k6 duration 120s + polling + teardown). Mark as the 7th test in the suite; do NOT parallelize (uses the shared test namespace).

- **Owner-ref Kind MUST be compared case-sensitively to `"AnalysisRun"` / `"Rollout"`.** These are the `Kind` values emitted by `parentOwnerRef` in `testrun.go:82,91`. No ambiguity.

- **Teardown must be robust to partial test failures.** If the test t.Fatals mid-sequence, teardown still runs and must clean up whatever subset of resources was created. Use `--ignore-not-found` on every delete, and delete ALL resource types (Rollout, AnalysisRun, AnalysisTemplate, Service, ConfigMap, TestRuns) to leave the namespace pristine for the next test run.

- **The new test does NOT verify that the plugin's Provider.Cleanup fired (Phase 11 coverage).** The ONLY assertion is kube-apiserver cascade GC behavior driven by AR deletion. If the metric plugin were still leaking (Phase 11 bug), the cascade assertion would still pass because kube-apiserver GC runs regardless of the plugin's own Cleanup. Phase 11's unit tests cover the Provider.Cleanup path; this e2e covers the kube-apiserver path. Both must hold.

    // getTestRunsOwnedBy returns the names of TestRun CRs in `namespace` whose
    // metadata.ownerReferences contain an entry matching ownerKind (and, if
    // ownerName is non-empty, also matching ownerName). The match is
    // case-sensitive on ownerKind -- use "AnalysisRun" or "Rollout" to mirror
    // the Kind strings parentOwnerRef emits in internal/provider/operator/testrun.go.
    func getTestRunsOwnedBy(cfg *envconf.Config, namespace, ownerKind, ownerName string) ([]string, error) {
        cmd := exec.Command("kubectl", "--kubeconfig", cfg.KubeconfigFile(),
            "get", "testruns", "-n", namespace, "-o", "json")
        out, err := cmd.Output()
        if err != nil {
            return nil, err
        }
        var list struct {
            Items []struct {
                Metadata struct {
                    Name            string `json:"name"`
                    OwnerReferences []struct {
                        Kind string `json:"kind"`
                        Name string `json:"name"`
                    } `json:"ownerReferences"`
                } `json:"metadata"`
            } `json:"items"`
        }
        if err := json.Unmarshal(out, &list); err != nil {
            return nil, err
        }
        var names []string
        for _, item := range list.Items {
            for _, ref := range item.Metadata.OwnerReferences {
                if ref.Kind != ownerKind {
                    continue
                }
                if ownerName != "" && ref.Name != ownerName {
                    continue
                }
                names = append(names, item.Metadata.Name)
                break
            }
        }
        return names, nil
    }

    // getTestRunStage returns status.stage of a named TestRun CR.
    // k6-operator v1.3.x stage enum: "initialization", "initialized", "created",
    // "started", "stopped", "finished", "error". There is no "running" literal.
    // The combined-canary test waits for "started" on both TestRuns before
    // issuing kubectl delete analysisrun.
    func getTestRunStage(cfg *envconf.Config, name, namespace string) (string, error) {
        cmd := exec.Command("kubectl", "--kubeconfig", cfg.KubeconfigFile(),
            "get", "testrun", name, "-n", namespace,
            "-o", "jsonpath={.status.stage}")
        out, err := cmd.Output()
        if err != nil {
            return "", err
        }
        return string(out), nil
    }

    // listAnalysisRuns returns the names of all AnalysisRun CRs in the namespace.
    func listAnalysisRuns(cfg *envconf.Config, namespace string) ([]string, error) {
        cmd := exec.Command("kubectl", "--kubeconfig", cfg.KubeconfigFile(),
            "get", "analysisruns", "-n", namespace,
            "-o", "jsonpath={.items[*].metadata.name}")
        out, err := cmd.Output()
        if err != nil {
            return nil, err
        }
        fields := strings.Fields(string(out))
        return fields, nil
    }

    // getTestRunLabel returns the value of a named label on a TestRun CR.
    func getTestRunLabel(cfg *envconf.Config, name, namespace, label string) (string, error) {
        // kubectl jsonpath requires escaping the dot/slash in label keys.
        jp := fmt.Sprintf("jsonpath={.metadata.labels.%s}", strings.NewReplacer(".", `\.`, "/", `\/`).Replace(label))
        cmd := exec.Command("kubectl", "--kubeconfig", cfg.KubeconfigFile(),
            "get", "testrun", name, "-n", namespace, "-o", jp)
        out, err := cmd.Output()
        if err != nil {
            return "", err
        }
        return string(out), nil
    }
