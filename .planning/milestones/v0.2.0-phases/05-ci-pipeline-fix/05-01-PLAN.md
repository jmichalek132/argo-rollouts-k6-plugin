<task type="auto">
  <name>Task 1: Fix Makefile DOCKER_HOST for cross-platform compatibility</name>
  <files>Makefile</files>
  <action>
Change DOCKER_HOST from hardcoded inline assignment to a conditional variable at the top of the Makefile. This is the CRITICAL prerequisite for D-02 (using `make test-e2e` in CI) -- without this fix, `make test-e2e` fails on GitHub Actions ubuntu-latest because the Colima socket path does not exist.

Specific changes:

1. Add a conditional DOCKER_HOST assignment near the top of the Makefile (after the existing LDFLAGS line):
   ```makefile
   # Default to Colima socket for local macOS dev; CI runners and standard Docker
   # installs leave DOCKER_HOST unset or set it to the standard /var/run/docker.sock.
   DOCKER_HOST ?= unix://$(HOME)/.colima/default/docker.sock
   ```

2. In the `test-e2e` recipe, change the inline DOCKER_HOST override to use the Makefile variable:
   ```makefile
   test-e2e:
   	GOPATH="$(HOME)/go" DOCKER_HOST="$(DOCKER_HOST)" \
   	go test -v -tags=e2e -count=1 -timeout=15m ./e2e/...
   ```

3. Apply the same change to `test-e2e-live`:
   ```makefile
   test-e2e-live:
   	GOPATH="$(HOME)/go" DOCKER_HOST="$(DOCKER_HOST)" \
   	K6_LIVE_TEST=true \
   	K6_CLOUD_TOKEN=$(K6_CLOUD_TOKEN) \
   	K6_STACK_ID=$(K6_STACK_ID) \
   	K6_TEST_ID=$(K6_TEST_ID) \
   	K6_FAILING_TEST_ID=$(K6_FAILING_TEST_ID) \
   	go test -v -tags=e2e -count=1 -timeout=30m ./e2e/...
   ```

The `?=` operator means: use the environment's DOCKER_HOST if set, otherwise default to the Colima path. On GitHub Actions, the e2e.yml sets `DOCKER_HOST: unix:///var/run/docker.sock` at the env level, which overrides the Makefile default.

Do NOT change the `test` target (unit tests do not need Docker).
Do NOT change the build targets.
Do NOT change the lint targets.
  </action>
