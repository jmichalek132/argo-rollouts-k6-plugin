<!-- Mock reachability architecture -->
From internal/provider/cloud/cloud.go:
- GrafanaCloudProvider has a `baseURL` field and `WithBaseURL(url string) Option` functional option
- When `baseURL` is set, `newK6Client()` overrides the k6 client's server config: `k6Cfg.Servers = k6.ServerConfigurations{{URL: p.baseURL}}`
- When `baseURL` is empty, the k6 client uses its default `https://api.k6.io`
- `metrics.go` also uses `p.baseURL` with fallback to `defaultBaseURL = "https://api.k6.io"`
- The `stackId` field is passed as an `X-Stack-Id` HTTP header (parsed as int32), NOT embedded in the URL path
- Both `cmd/metric-plugin/main.go` and `cmd/step-plugin/main.go` call `cloud.NewGrafanaCloudProvider()` with no options

The mock server approach uses Docker host-gateway networking + K6_BASE_URL env var on the
argo-rollouts controller pod. This reads K6_BASE_URL in cmd/*/main.go (NOT internal/) and
passes it to the existing WithBaseURL() option. No changes to internal/ are needed.

4. **Mock server reachability from kind cluster -- Docker host-gateway networking (locked approach):**

   The GrafanaCloudProvider constructs API calls to `baseURL` (default `https://api.k6.io`). The `WithBaseURL(url)` functional option on `GrafanaCloudProvider` already overrides this base URL (see `cloud.go` line 26-29 and `newK6Client()` line 49-51). The plugin binaries (`cmd/metric-plugin/main.go` and `cmd/step-plugin/main.go`) currently call `cloud.NewGrafanaCloudProvider()` with no options.

   **Approach:** Add a `K6_BASE_URL` env var check in both `cmd/metric-plugin/main.go` and `cmd/step-plugin/main.go`. This is a one-line change to each file (in `cmd/`, NOT `internal/` -- respects the phase boundary). The env var is read at binary startup and passed to the existing `WithBaseURL()` option:

   In both `cmd/metric-plugin/main.go` and `cmd/step-plugin/main.go`, change:
   ```go
   p := cloud.NewGrafanaCloudProvider()
   ```
   to:
   ```go
   var opts []cloud.Option
   if baseURL := os.Getenv("K6_BASE_URL"); baseURL != "" {
       opts = append(opts, cloud.WithBaseURL(baseURL))
   }
   p := cloud.NewGrafanaCloudProvider(opts...)
   ```

   This pattern mirrors the existing `LOG_LEVEL` env var already read in both binaries. No changes to `internal/` are needed -- `WithBaseURL` already exists in `cloud.go`.

   In the e2e TestMain, the mock server URL is injected into the argo-rollouts controller deployment via:
   a. Start mock server on `0.0.0.0:{PORT}` on the host
   b. Discover the Docker bridge gateway IP: run `docker network inspect kind -f '{{(index .IPAM.Config 0).Gateway}}'` and fall back to `172.18.0.1` (the default kind network gateway) if the command fails
   c. Set `K6_BASE_URL=http://{GATEWAY_IP}:{PORT}` as an env var on the argo-rollouts controller deployment: `kubectl set env deployment/argo-rollouts -n argo-rollouts K6_BASE_URL=http://{GATEWAY_IP}:{PORT}`
   d. The controller restarts, plugin binaries launched by the controller inherit this env var, and `WithBaseURL()` routes all API calls to the mock server

   This requires NO changes to `internal/provider/cloud/cloud.go` or any file under `internal/`.
