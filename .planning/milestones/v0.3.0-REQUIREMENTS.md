# Requirements Archive: v0.3.0 In-Cluster Execution

**Archived:** 2026-04-20 at milestone close.
**Defined:** 2026-04-15
**Core Value:** Rollouts automatically pass or roll back based on real load test results — no manual gates, no guesswork.

All 16 v0.3.0 requirements shipped. Three decimal phases (08.1, 08.2, 08.3) were inserted post-Phase-10 to fix bugs surfaced by the e2e suite itself; they are not mapped to REQ-IDs because they are stability fixes for the same requirements, not new ones.

## Notes on Outcomes

**K6OP-08 scope at shipping:** Cancellation-path cleanup (Terminate/Abort) is fully implemented via `StopRun`. Success-path cleanup (post-terminal delete of the TestRun CR + runner pods) is deferred after Phase 08.3 — k6-operator's `spec.Cleanup=post` was removed because it races the plugin's status-read loop. A future phase should implement `GarbageCollect` on the metric plugin and a symmetric post-terminal delete on the step plugin. Documented in `.planning/phases/08.3-.../08.3-01-PLAN.md <followup_not_in_scope>`.

**Decimal phases inserted:** 08.1 (metadata wiring), 08.2 (parallelism=1 default), 08.3 (unset spec.Cleanup). These were emergent from the e2e suite itself — each fix unblocked the next layer and exposed a new hidden bug. Not a pattern to repeat, but the `/gsd-debug` → fix-as-phase → verify cycle worked well for capturing the debugging trail.

## Out of Scope (Validated)

Items that remained out of scope through v0.3.0. No changes from v0.3.0 definition.

| Feature | Reason |
|---------|--------|
| Local binary execution (subprocess) | Anti-feature: corrupts go-plugin stdout protocol (grafana/k6#3744), unbounded resource consumption in controller pod, no process isolation. k6-operator covers all in-cluster use cases safely. |
| Inline script in plugin config | YAML escaping nightmares, no syntax highlighting, no version control. ConfigMap is the right abstraction. |
| Real-time metric streaming from in-cluster k6 | End-of-test handleSummary JSON is sufficient. For real-time, users configure `k6 --out prometheus` and use Argo Rollouts' Prometheus metric provider. |
| VU/duration override in plugin config | Creates divergence from k6 script definition — same rationale as v1.0. |
| Automatic k6-operator installation | User's responsibility. Plugin fails gracefully with clear error if CRDs missing. |
| Distributed execution without k6-operator | Reimplementing execution segments is exactly what k6-operator does. Use k6-operator provider. |
| PersistentVolume script sourcing | ConfigMaps cover 95%+ of k6 scripts (< 1 MiB). Users needing PVC can use k6-operator directly. |
