**Create internal/provider/operator/exitcode.go**

Package `operator`. Contains:

1. Exit code constants (per RESEARCH.md, cited from k6 source):
```go
const (
    exitCodeSuccess          = 0
    exitCodeThresholdsFailed = 99
)
```

2. `exitCodeToRunState(code int32) provider.RunState` -- per D-05:
   - 0 -> provider.Passed
   - 99 -> provider.Failed (thresholds breached)
   - any other non-zero -> provider.Errored (script error, infra issue)

3. `stageToRunState(stage string) provider.RunState` -- per RESEARCH.md stage mapping:
   - "initialization", "initialized", "created", "started" -> provider.Running
   - "stopped" -> provider.Running (stopping but not finished)
   - "finished" -> provider.Running (IMPORTANT: returns Running because exit code check is needed to determine Passed/Failed -- this is the issue #577 workaround)
   - "error" -> provider.Errored
   - "" (empty string) -> provider.Running (addresses MEDIUM review concern: absent stage field on freshly created CR returns Running instead of error)
   - default (unknown) -> provider.Running (keep polling)

4. `checkRunnerExitCodes(ctx context.Context, client kubernetes.Interface, ns, testRunName string) (provider.RunState, error)`:
   - Label selector: `fmt.Sprintf("app=k6,k6_cr=%s,runner=true", testRunName)` (per RESEARCH.md, labels from k6-operator runner.go)
   - List pods with `client.CoreV1().Pods(ns).List(ctx, metav1.ListOptions{LabelSelector: selector})`
   - If len(pods.Items) == 0 -> return provider.Errored, fmt.Errorf("no runner pods found for TestRun %s", testRunName)

   Addresses HIGH review concern about checkRunnerExitCodes edge cases:

   - **Zero pods**: Return error. This is the correct behavior because k6-operator creates runner pods before setting stage to "finished". If no pods are found, it means either (a) they were garbage collected early or (b) we're hitting a race. Either way, the error bubbles up and GetRunResult treats it as Errored with a warning log.

   - **Restarted containers**: If ANY container has RestartCount > 0, return provider.Errored immediately. k6-operator sets backoff limit to 0 (issue #75), so restarts should not happen normally. A restart indicates an infrastructure problem. This addresses the review concern about "restarted runner containers with prior exit codes."

   - **Pods not yet terminated**: If ANY container has Terminated == nil, return provider.Running. The next poll cycle will retry.

   - **Multiple containers per pod**: Iterate ALL ContainerStatuses per pod, not just the first. Exit code precedence: Errored > Failed > Passed.

   ```go
   func checkRunnerExitCodes(ctx context.Context, client kubernetes.Interface, ns, testRunName string) (provider.RunState, error) {
       selector := fmt.Sprintf("app=k6,k6_cr=%s,runner=true", testRunName)
       pods, err := client.CoreV1().Pods(ns).List(ctx, metav1.ListOptions{
           LabelSelector: selector,
       })
       if err != nil {
           return provider.Errored, fmt.Errorf("list runner pods: %w", err)
       }
       if len(pods.Items) == 0 {
           return provider.Errored, fmt.Errorf("no runner pods found for TestRun %s", testRunName)
       }

       worst := provider.Passed
       for _, pod := range pods.Items {
           for _, cs := range pod.Status.ContainerStatuses {
               // Restarted container = infrastructure problem (k6-operator sets backoff limit 0).
               // Addresses review concern: restarted containers with prior exit codes.
               if cs.RestartCount > 0 {
                   slog.Warn("runner container restarted unexpectedly",
                       "pod", pod.Name,
                       "container", cs.Name,
                       "restartCount", cs.RestartCount,
                       "testRunName", testRunName,
                   )
                   return provider.Errored, nil
               }
               // Pod still running -- retry on next poll (per Pitfall 2).
               if cs.State.Terminated == nil {
                   return provider.Running, nil
               }
               state := exitCodeToRunState(cs.State.Terminated.ExitCode)
               if state == provider.Errored {
                   return provider.Errored, nil
               }
               if state == provider.Failed {
                   worst = provider.Failed
               }
           }
       }

       slog.Debug("checked runner pod exit codes",
           "testRunName", testRunName,
           "namespace", ns,
           "podCount", len(pods.Items),
           "result", worst,
       )
       return worst, nil
   }
   ```
