**Wired Rollout metadata extraction (Namespace/RolloutName/RolloutUID) into `internal/step/step.go` Run, Terminate, and Abort via a shared `populateFromRollout` helper and an extended `stopActiveRun` signature — so step-triggered k6-operator TestRun CRs now carry the Rollout OwnerReference and target the correct namespace.**

## Accomplishments

- `step.go` Run, Terminate, and Abort now bind the `*v1alpha1.Rollout` argument (previously `_`) and extract metadata before provider dispatch.
- `populateFromRollout(cfg, rollout, phase)` helper added in `step.go`:
  - `cfg.RolloutName = rollout.Name` (D-06: Rollout IS the parent; no owner-ref walk)
  - `cfg.RolloutUID = string(rollout.UID)` (D-05: populated by `parentOwnerRef` for GC)
  - `cfg.Namespace = rollout.Namespace` only when `cfg.Namespace == ""` (D-01: user cfg wins)
  - `cfg.AnalysisRunName`/`cfg.AnalysisRunUID` intentionally untouched (D-05: step has no AR)
  - Nil Rollout: `slog.Warn("nil Rollout received; skipping metadata extraction", "phase", phase)` and return (D-09: gob-boundary defense)
- `stopActiveRun` signature extended to `(rollout *v1alpha1.Rollout, ctx *types.RpcStepContext, action string)`. Terminate and Abort pass `rollout` through, so StopRun now receives a fully-populated `cfg` symmetric with TriggerRun.
- Test fixture `testRollout(name, ns, uid)` added; three new tests added.
- All 28 pre-existing step tests continue to pass unchanged (D-08 preserved) — they exercise the nil-Rollout warn+skip path that didn't exist before.

## Decisions Made

- **Extend `stopActiveRun` signature rather than duplicate the extraction block.** Terminate and Abort are the only callers; a signature change keeps metadata extraction behind a single helper call and guarantees the stop path stays symmetric with the trigger path. An inline copy would have created drift risk the next time the extraction rules change.
- **Use `k8stypes` alias for `k8s.io/apimachinery/pkg/types`.** The test file already imports `github.com/argoproj/argo-rollouts/utils/plugin/types` under the default name `types`. Aliasing the k8s types import avoids identifier collision and keeps the plugin `types.RpcStepResult` / `types.PhaseFailed` references in existing tests untouched.
- **Per-method phase string in `populateFromRollout` call.** `"step.Run"` and `"step."+action` (where action is "terminate"/"abort") give log grep-ability if the warn branch fires in production, without complicating the helper signature.
- **Helper placed in the same file as its callers.** Single consumer, unexported, lives next to `parseConfig` where it is used. No cross-package surface, no new file.

## Revision Notes Applied (from codex review)

- **R6 (MEDIUM verification gap):** `TestTerminate_PassesPopulatedConfigToStopRun` directly asserts that `Terminate → stopActiveRun → StopRun` propagates `cfg.Namespace`, `cfg.RolloutName`, and `cfg.RolloutUID`. Proves the signature change on `stopActiveRun` actually delivers metadata on the stop path, not just on Run. Since Abort uses the same shared helper with the same code path, Terminate verification transitively covers Abort.
- **R7 (behavior over grep count):** The plan dropped the brittle `grep -c '_ \*v1alpha1.Rollout'` check in favor of the behavioral check above. The three new tests are the authoritative verification that the `rollout` parameter is actually bound and used — a grep count cannot prove the extraction ran and populated cfg.
- **R8 (nil-Rollout graceful degradation note):** `populateFromRollout` doc comment explicitly documents that nil Rollout preserves pre-fix behavior (cfg retains whatever values parseConfig produced plus the operator's `"default"` namespace safety net at the provider layer), and that downstream StopRun behavior with under-populated cfg is provider-specific and out of scope for this bug fix.
