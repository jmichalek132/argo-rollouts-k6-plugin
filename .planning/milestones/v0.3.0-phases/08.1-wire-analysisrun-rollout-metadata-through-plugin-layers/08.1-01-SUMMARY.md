# Phase 08.1 Plan 01: RolloutUID field + parent owner-ref precedence Summary

**Added `PluginConfig.RolloutUID` and refactored the k6-operator owner-ref helper into `parentOwnerRef(cfg)` that encodes `AR > Rollout > none` precedence so Wave 2 step.go/metric.go wiring can rely on a single source of truth for CR ownership.**

## Accomplishments

- `PluginConfig.RolloutUID string` field added with `json:"-"` tag and styling aligned to its three sibling injected fields (RolloutName, AnalysisRunName, AnalysisRunUID).
- `analysisRunOwnerRef(name, uid)` replaced by `parentOwnerRef(cfg *provider.PluginConfig)` which encodes D-07 precedence: AnalysisRun wins when `cfg.AnalysisRunUID != ""`; else Rollout wins when `cfg.RolloutUID != ""`; else nil.
- Both call sites (`buildTestRun` and `buildPrivateLoadZone`) now use the new helper — no dangling references to the old helper remain.
- Helper doc comment documents the caller invariant (R2 from codex review): caller MUST set both UID and matching Name for the chosen parent.
- Two new operator tests verify the Rollout-owned branch and AR-wins-over-Rollout precedence; both existing owner-ref tests continue to pass unmodified.

## Decisions Made

- **Helper signature takes `*provider.PluginConfig` rather than four strings.** Future-proofs adding more parent kinds (e.g. Experiment) without touching call sites, and matches the existing stateless-cfg pattern used across the provider package.
- **Invariant comment landed verbatim (R2).** The doc comment explicitly documents that callers must set the matching Name for the chosen UID. If a caller sets only UID, the Kubernetes API server rejects the Create — surfacing bugs loudly rather than silently dropping the ownership link.
- **Old `analysisRunOwnerRef` fully removed.** Not kept as a deprecated alias because (a) no out-of-package callers (unexported), and (b) any in-package caller must flow through the precedence logic.
