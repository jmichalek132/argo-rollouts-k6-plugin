### `internal/provider/operator/operator.go` (service, data transform)

**Analog:** self — the existing owner-ref construction in `buildTestRun`/`buildPrivateLoadZone` via the `analysisRunOwnerRef(name, uid)` helper in `testrun.go` (L68–82). The TriggerRun branch at L230–254 is the site being refactored per D-07.

**Current owner-ref helper to refactor** — `internal/provider/operator/testrun.go` L68–82:
```go
// analysisRunOwnerRef creates an OwnerReference for the AnalysisRun (per D-09).
// If analysisRunUID is empty, returns nil (fallback to label-based identification).
// Both Name and UID are required: the Kubernetes API server validates that Name is
// non-empty in OwnerReferences, and UID is used for the actual GC lookup.
func analysisRunOwnerRef(analysisRunName, analysisRunUID string) []metav1.OwnerReference {
	if analysisRunUID == "" {
		return nil
	}
	return []metav1.OwnerReference{{
		APIVersion: "argoproj.io/v1alpha1",
		Kind:       "AnalysisRun",
		Name:       analysisRunName,
		UID:        types.UID(analysisRunUID),
	}}
}
```

**Call sites in `testrun.go`** — L138, L180:
```go
OwnerReferences: analysisRunOwnerRef(cfg.AnalysisRunName, cfg.AnalysisRunUID),
```

**D-07 refactor pattern** — extend the helper to encode the AR-wins-over-Rollout precedence. Suggested name per CONTEXT.md Claude's Discretion: `parentOwnerRef(cfg)` keeping the AR helper's early-return style:
```go
// parentOwnerRef returns the OwnerReference chain for a k6-operator CR per D-07.
// Precedence: AnalysisRun > Rollout > none.
func parentOwnerRef(cfg *provider.PluginConfig) []metav1.OwnerReference {
	if cfg.AnalysisRunUID != "" {
		return []metav1.OwnerReference{{
			APIVersion: "argoproj.io/v1alpha1",
			Kind:       "AnalysisRun",
			Name:       cfg.AnalysisRunName,
			UID:        types.UID(cfg.AnalysisRunUID),
		}}
	}
	if cfg.RolloutUID != "" {
		return []metav1.OwnerReference{{
			APIVersion: "argoproj.io/v1alpha1",
			Kind:       "Rollout",
			Name:       cfg.RolloutName,
			UID:        types.UID(cfg.RolloutUID),
		}}
	}
	return nil
}
```
The two call sites at `testrun.go` L138 and L180 then become `OwnerReferences: parentOwnerRef(cfg),`.

**Kind/APIVersion fidelity** — both Rollout and AnalysisRun live under `argoproj.io/v1alpha1` (verified by the existing helper using `"argoproj.io/v1alpha1"` for AnalysisRun). Use the same APIVersion for the Rollout case; Kind is `"Rollout"`.

**No changes needed** in `operator.go` `TriggerRun` itself — the logic already flows through the build functions; only the helper they both call gets extended.

**Namespace-fallback block to leave as-is per D-02** — `operator.go` L145–148 and L217–221:
```go
ns := cfg.Namespace
if ns == "" {
	ns = "default"
}
```
Preserved as defensive safety net.

## Shared Patterns

### Slog structured logging
**Source:** across `metric.go`, `step.go`, `operator.go`, `cloud.go`
**Apply to:** all new warning paths added in metric.go and step.go

Canonical form: `slog.Warn(<message>, <key1>, <value1>, <key2>, <value2>, ...)` with key names in camelCase. Recurring field names to reuse for consistency:
- `"runId"`, `"namespace"`, `"name"`, `"error"` — already established
- `"provider"` (operator package adds `"provider", p.Name()`) — not applicable to metric/step layer

New conventions for Phase 08.1 (Claude's Discretion per CONTEXT.md):
- `"analysisRun"` for AR.Name (parallels the existing `"runId"` / `"name"` pattern)
- `"rollout"` for Rollout.Name
- `"phase"` for which method generated the warning (e.g. `"metric.Run"`, `"step.Terminate"`) — only if the call site is ambiguous

### `json:"-"` injected fields pattern
**Source:** `internal/provider/config.go` L42–44
**Apply to:** the new `RolloutUID` field

All four injected fields (`RolloutName`, `AnalysisRunName`, `AnalysisRunUID`, new `RolloutUID`) share the same comment-and-tag shape. Keep identical spacing and inline-comment style so the block stays visually aligned.

### Owner-ref construction with typed UID
**Source:** `internal/provider/operator/testrun.go` L72–82 (`analysisRunOwnerRef`)
**Apply to:** the extended `parentOwnerRef(cfg)` helper

```go
UID: types.UID(uidString)    // import "k8s.io/apimachinery/pkg/types"
```
Both the existing AR owner-ref and the new Rollout owner-ref must wrap the plain-string UID with `types.UID(...)`. `APIVersion` is `"argoproj.io/v1alpha1"` for both AR and Rollout. Empty-UID short-circuit (`return nil`) remains the fallback that produces no owner reference.

### Closure-based provider mock
**Source:** `internal/provider/providertest/mock.go` — `MockProvider` with `TriggerRunFn`/`GetRunResultFn`/`StopRunFn` function fields
**Apply to:** all new tests in `metric_test.go` and `step_test.go`

Set only the function field(s) your test exercises; leave the rest nil. Inside the closure, assert on the received `*provider.PluginConfig` to verify plumbing. This is the single supported assertion path for metadata population because the Provider interface (`TriggerRun(ctx, cfg)`) doesn't expose the AR/Rollout directly — the plugin layer must have already encoded the metadata into `cfg`.

### Fake dynamic client + decode+Get pattern for owner-ref assertions
**Source:** `internal/provider/operator/operator_test.go` L286–317
**Apply to:** the two new `operator_test.go` tests

```go
runID, err := p.TriggerRun(context.Background(), cfg)
require.NoError(t, err)
ns, _, name, decErr := decodeRunID(runID)
require.NoError(t, decErr)
created, getErr := fakeDyn.Resource(testRunGVR).Namespace(ns).Get(
    context.Background(), name, metav1.GetOptions{},
)
require.NoError(t, getErr)
ownerRefs := created.GetOwnerReferences()
```
This round-trips through the real code path (TriggerRun → buildTestRun → dynClient.Create → fake store) rather than unit-testing the helper alone, which is why the existing AR tests live here and the new Rollout tests should too.
