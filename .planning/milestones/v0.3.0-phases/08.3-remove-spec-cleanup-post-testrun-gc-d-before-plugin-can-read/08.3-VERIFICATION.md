## Follow-up (outside phase 08.3 scope)

Success-path TestRun/pod cleanup now relies on manual sweeps or namespace churn — the plugin creates TestRun CRs and never deletes them on success. A future phase should:

- Implement `GarbageCollect` on the metric plugin (called by Argo Rollouts after terminal analysis) to delete the TestRun CR and its pods.
- Add a symmetric post-terminal delete on the step plugin's terminal state handler so successful step runs don't leak resources.
- Consider a sweep job / TTL controller as defense-in-depth if plugins crash before reaching GarbageCollect.

`StopRun` already handles Terminate/Abort paths correctly — only the success path leaks.
